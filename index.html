// How it works

Most agents summarize. This one evaluates.

Six interlocking rule areas decide whether a workflow is governed, traceable, and audit-ready — then build the connected space to prove it.

DG-01

Baseline Governance

Classify data, map its lifecycle, define access, track retention, collect evidence.

CP-01

Compliance Program

Consent, security controls, policies, training, third-party risk, ownership.

RO-01

Regulatory Overlays

GDPR, CCPA/CPRA, HIPAA, and PCI-DSS trigger logic and required controls.

PL-01

Lifecycle Mapping

Connect initiation → closure to the documents and Gantt phases each requires.

DC-01

Document Governance

Link SOPs to tasks, detect contradictions, version artifacts, enforce source of truth.

AE-01

Agent Evaluation

Cite rule IDs, flag missing evidence, surface assumptions, define escalation.

// Intelligent intake

It doesn't guess the gaps. It asks the right person.

When the framework can't resolve something, it generates a precise question and routes it to the role who owns that answer — so documentation gets filled by the right people, and execs stay out of the weeds.

// Meet the team where it is

How lean is the team really — and where can people grow?

Before prioritizing projects, the framework reads how the Agile team is actually structured — which roles are doubled up, which are thin, and where someone holds a seat they were never equipped for.

Not to grade anyone. To prioritize realistically and give people a path to the technical footing they're missing.

The point isn't to expose a thin team — it's to run one well. The framework meets the org where it is, keeps people out of decisions they're not equipped for, and turns gaps into growth paths.
